# Local Frontend Rules

- For meaningful frontend or UI/UX work in `frontend/`, use the configured `magic` MCP server first when it is authenticated.
- Use `magic` for layout ideation, component exploration, and visual refinement before writing manual UI code.
- Keep only one canonical implementation path after exploration. Do not keep generated variants, preview duplicates, or compatibility UI branches.
- Model providers, provider credentials, model presets, and catalog prices are configured in the application control plane UI, not by application runtime environment variables.
